You are taking the role of a knowledge engineer that has to extract the ontology alignments between two ontologies, that will be referred to in the prompt as "Ontology 1", and "Ontology 2". - 3: From the first ontology you will add the entities from Ontology 1 to the "subject_id" column, and their labels (rdfs:label) to the "subject_label" column.

	- 4.3: Look for semantic matches with a similarity threshold <= 0.5 within the Ontology 2. Add to the "mapping_justification" semapv:SemanticSimilarityThresholdMatching. 
	- 4.4: Look for matches where the entities from Ontology 1 and Ontology 2 have the same instances. Add to the "mapping_justification" semapv:InstanceBasedMatching. 
	- 4.5: Then perform a logical reasoning process tho identify matches within Ontology 2. Add to the "mapping_justification" semapv:LogicalReasoning. 
	- 4.6: Then analyse the ontological structure of an entity and find matches within Ontology 2 that follow a similar structure. Add to the "mapping_justification" semapv:StructuralMatching. 
	- 4.7: After finding all matches from the methods above perform a matching process by chaining the marches between other entities using the transitive property. Add to the "mapping_justification" semapv:MappingChaining. 
	- 4.8: If there are any other matches from other kind of knowledge.  Add to the "mapping_justification" semapv:BackgroundKnowledgeBasedMatching. 
- 5. For any found matches a mapping predicate has to be selected depending on how it has matched and added to "predicate_id":

	- B): owl:equivalentClass if subject and object are owl classes and are the same.
	- C): owl:equivalentProperty if the subject and object are properties and the two properties are the same.
	- D): rdfs:subClassOf if the subject and object are owl classes and subject is subclass of object.
	- E): rdfs:subPropertyOf if subject and object are owl properties and subject is subproperty of the object.
	- F) skos:relatedMatch if subject and object are associated in some unspecified way
	- G) skos:closeMatch if subject and object are sufficiently similar so that they can be used interchangeably. 
	- H) skos:exactMatch if subject and object can with a high degree of confidence be used interchangeably.
	- I) skos:narrowMatch if the object of the triple is narrower than the subject.
	- J) skos:broadMatch if the object of the triple is broader than the subject.
	- K) oboInOwl:hasDbXref if the two terms are related in some way.
	- L) rdfs:seeAlso if the subject and object are related in an unspecified way
